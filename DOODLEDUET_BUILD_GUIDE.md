# DoodleDuet CasaOS App - Build & Deploy Quick Start

## Overview

This guide walks you through generating the app icon, validating the app configuration, and deploying it to your CasaOS appstore.

## Quick Start (Windows)

### Step 1: Generate Icon & Validate

Run the PowerShell build script:

```powershell
# From your vintagecontainer directory
.\build-and-deploy-doodleduet.ps1
```

This will:
- ✓ Generate the DoodleDuet icon (512x512 PNG)
- ✓ Validate docker-compose.yml
- ✓ Validate store.json JSON syntax
- ✓ Check file structure
- ✓ Test Docker configuration
- ⚠ Optionally commit and push to GitHub

### Step 2: Update store.json

The script will show you if the app entry is missing from `store.json`. If so:

1. Open `store.json` in your editor
2. Find the `"apps": [` array
3. Add this entry (keep entries alphabetical or in order):

```json
{
  "name": "doodleduet-app",
  "title": { "en_us": "DoodleDuet" },
  "icon": "https://raw.githubusercontent.com/JoshEvans-Weber/casaos-store/main/Apps/doodleduet-app/icon.png",
  "tagline": { "en_us": "Real-time collaborative drawing application with live chat" },
  "description": { "en_us": "DoodleDuet is a real-time collaborative drawing application built with Next.js 15, Socket.IO, and Redis. Draw together, chat in real-time, and create amazing artwork with friends! Features include real-time collaboration, room management, multiple drawing tools, infinite canvas with pan/zoom, live chat, admin controls, responsive design for desktop and mobile, user avatars, and presence indicators." },
  "category": "Games",
  "version": "1.0.0",
  "updated": "2026-03-22T06:30:00",
  "image": { "amd64": "aiyuayaan/doodleduet-app:v1.0.0" }
}
```

### Step 3: Commit & Push to GitHub

After running the script, you'll be prompted to commit and push. If you skipped it:

```bash
cd /path/to/your/casaos-store
git add Apps/doodleduet-app/
git commit -m "Add DoodleDuet collaborative drawing app"
git push
```

### Step 4: Test in CasaOS

1. Log into your CasaOS instance
2. Go to **App Store**
3. Click **Refresh** to pull the latest app definitions
4. Search for **"DoodleDuet"**
5. Click **Install**
6. Wait for containers to start (redis + doodleduet-app)
7. Click the app or go to `http://your-casa-ip:3000`
8. Test creating a drawing room and inviting someone

## Quick Start (Linux/Mac)

If you're on a Linux or Mac system:

```bash
# Make script executable
chmod +x build-and-deploy-doodleduet.sh

# Run the script
./build-and-deploy-doodleduet.sh
```

## Manual Steps (If Scripts Fail)

### 1. Generate Icon

```bash
# Install PIL if needed
pip3 install pillow

# Generate icon
python3 generate_icon.py
```

This creates: `Apps/doodleduet-app/icon.png`

### 2. Validate Configuration

```bash
# Check docker-compose.yml syntax
docker-compose -f Apps/doodleduet-app/docker-compose.yml config

# Check JSON validity
python3 -c "import json; json.load(open('store.json'))" && echo "✓ Valid JSON"
```

### 3. Manual Git Commit

```bash
git add Apps/doodleduet-app/
git commit -m "Add DoodleDuet collaborative drawing app to CasaOS store"
git push
```

## Troubleshooting

### Icon Generation Failed

**Error**: `ModuleNotFoundError: No module named 'PIL'`

**Solution**:
```bash
pip3 install pillow
python3 generate_icon.py
```

### docker-compose Config Invalid

**Error**: Invalid format or missing fields

**Check**:
1. YAML syntax is correct (quotes, indentation)
2. All required x-casaos fields present
3. Service names match container_name
4. Port numbers are correct

**Validate**:
```bash
python3 -m yaml < Apps/doodleduet-app/docker-compose.yml
```

### store.json Invalid

**Error**: JSON parsing error

**Solution**:
```bash
# Check syntax
python3 -m json.tool store.json

# Fix manually - check for:
# - Missing commas between entries
# - Trailing commas
# - Unescaped quotes
# - Non-matching braces
```

### App Won't Install in CasaOS

**Check**:
1. Icon URL is accessible: `https://raw.githubusercontent.com/JoshEvans-Weber/casaos-store/main/Apps/doodleduet-app/icon.png`
2. Docker image exists: `aiyuayaan/doodleduet-app:v1.0.0`
3. Port 3000 isn't already in use
4. Redis container started successfully

**Debug**:
```bash
# Check CasaOS logs
docker logs casa-app-management

# Check app-specific logs
docker logs doodleduet-app
docker logs doodleduet-redis
```

### Port Conflict

If port 3000 is already in use, modify `docker-compose.yml`:

```yaml
ports:
  - "0.0.0.0:3001:3000/tcp"  # Map 3001 -> 3000
```

Then update the icon URL to match the new port in any bookmarks.

## Environment Variables

No additional configuration needed! The app uses:

- `REDIS_URL`: Automatically set to `redis://redis:6379`
- `NODE_ENV`: Set to `production`

### Optional Customizations

If you fork the Docker image, you can add:

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - NODE_ENV=production
  - MAX_ROOM_SIZE=10  # Max users per room
  - AUTO_CLEANUP_MINUTES=60  # Delete inactive rooms after X minutes
```

## Advanced: Building Your Own Image

If you want to build a custom Docker image for DoodleDuet:

1. Clone the [DoodleDuet GitHub repo](https://github.com/aiyuayaan/doodleduet)
2. Build: `docker build -t your-registry/doodleduet-app:v1.0.0 .`
3. Push: `docker push your-registry/doodleduet-app:v1.0.0`
4. Update `docker-compose.yml` and `store.json` with your image URL

## Updating the App

When a new version is released:

1. Update version in `docker-compose.yml` (x-casaos)
2. Update version in `store.json`
3. Update `updated` timestamp in `store.json`
4. Commit and push

Example:
```bash
# In store.json
"version": "1.0.1",
"updated": "2026-03-23T10:00:00",
"image": { "amd64": "aiyuayaan/doodleduet-app:v1.0.1" }
```
